# SSoT 문서 형식 규약

모든 문서는 아래 frontmatter + 본문 구조를 따른다. 스크립트와 사람 모두 이 규약을 지킨다.

```markdown
---
기술지원명: <전체 내용을 한 줄로 요약한 제목>
카테고리: <디자인|보안|개발|API|관리|계약 중 하나 — 예외 없음>
작성자: <Figma 또는 단군소프트 담당자 이름>
승인자: <PR 승인자 — PR 머지 시 자동 기입, 그 전까지 "(승인 대기)">
출처: <메일 제목 또는 Figma 헬프페이지 제목>
출처링크: <헬프페이지 URL — 메일이면 생략 가능>
게시일: <PR 승인일 — PR 머지 시 자동 기입, 그 전까지 "(승인 대기)">
검토일: <최종 검토일 — 게시일과 동일하게 시작, 큐레이터가 검토 시 갱신>
---

## 내용

<기술지원 내용 요약. 문제 상황 → 원인 → 해결 방법 순서.>

## 메일 초안 (메일 출처인 경우)

<유사 문의에 재사용할 수 있는 답변 초안>
```

## 규칙

1. 카테고리는 6개 중 하나만. 애매하면 가장 가까운 것을 강제 선택 (예외 없음)
2. 메일 출처 문서는 고객 개인정보(이름, 이메일, 전화번호, 계약 금액)를 마스킹한다
3. 작성자 = 출처 기준: 헬프페이지 → `Figma`, 메일 → 메일을 주고받은 단군소프트 담당자
4. 승인자/게시일은 스크립트가 건드리지 않는다 — PR 머지 워크플로가 자동 기입
5. 검토일이 90일을 초과한 문서는 매월 자동으로 이슈에 리스트업된다
