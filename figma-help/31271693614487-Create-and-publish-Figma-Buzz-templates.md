## 내용

Figma Buzz was released in [open beta](https://help.figma.com/hc/en-us/articles/4406787442711) at Config 2025. It is currently available on all plans and seat types. [Learn more about what’s included in the beta.](https://help.figma.com/hc/en-us/articles/4406787442711)

Who can use this feature

Who can use this feature

While in beta, Figma Buzz is available on all seats and plans.

Publishing templates is available on all paid plans. Professional teams can create five team templates.

Create custom Figma Buzz templates and publish them to your team or organization.

## Create a new template

Start with a new file where you’ll build a template.

1. [Create a new Buzz file](https://help.figma.com/hc/en-us/articles/360038511153).
2. From the template picker, click **New template**.

![Template picker interface in Figma Buzz with cursor over "New template" button for creating a new design.](https://help.figma.com/hc/article_attachments/31881277439511)

Once you’ve created a new file, you can either:

- [Edit your asset](https://help.figma.com/hc/articles/31271798141975) directly in Figma Buzz
- Paste existing designs from Figma Design into your Buzz file

## Make your template customizable

### Use component properties to create configurable templates

To offer configurable templates connected to your design system, build your template with component sets and component properties.

You can use component properties to define:

- Layout options (such as size, ratio, or color)
- Show/hide controls
- Replaceable visuals

[Learn more about building templates with component properties](https://help.figma.com/hc/en-us/articles/35659298120215).

### Use variable modes

Templates that include [variable modes](https://help.figma.com/hc/en-us/articles/15339657135383-Guide-to-variables-in-Figma) from Figma Design can be used in Figma Buzz.

Variable modes allow template creators to define configurations (such as color themes, localized content, or other options) within a single template.

[Learn more about building templates with variable modes](https://help.figma.com/hc/en-us/articles/35659298120215).

## Name layers

You can edit layer names for text in your assets to give consumers of your template guidance on what content it should be used for.

1. Select an asset.
2. Toggle to  [use Figma Design tools in Buzz](https://help.figma.com/hc/en-us/articles/31271851622807).
3. From the **Layers** panel on the bottom left, edit layer names and lock their position.

## Lock objects

Lock objects before publishing a template to keep anyone using it on brand, or prevent them from making unwanted edits.

Locking objects will impact what users see when they use your template.

- When someone uses your template, they will only see unlocked object fields available for editing in their [**Edit content**](https://help.figma.com/hc/articles/31271733056023) tab.
- To edit locked objects, someone using your template can click  to [remove editing restrictions](https://help.figma.com/hc/articles/31271733056023) from the template.
- If a user switches to [design mode](https://help.figma.com/hc/articles/31271851622807), locked objects will be indicated in the **Layers** panel, and can be unlocked by clicking the  lock icon.

To lock objects in your assets:

1. Select the object in asset.
2. Click  **Lock position**.

![Cursor hovers over "lock position" icon on toolbar above the second asset.](https://help.figma.com/hc/article_attachments/31918776614423)

## Publish a template

Once you’ve prepared your file, you’re ready to publish.

When you publish a template, you will be able to review and select which assets in the file will be published as templates.

Note: You cannot publish a template file from your drafts. Before you publish, [move the file](https://help.figma.com/hc/en-us/articles/360038511573-Move-a-file) from your drafts folder to a project.

### From Share modal

1. Click **Share**.
2. Select **Publish template**.
3. Enter in template details, including a name, thumbnail, and description.
4. Click **Publish**.

### From file menu

1. Click the  dropdown menu next to the file name.
2. Select **Publish template**.
3. Enter in template details, including a name, thumbnail, and description.
4. Click **Publish**.

## Edit a published template

1. Open the original template file.
2. Make your changes to the template.
3. Click **Share**.
4. Select **Update template**.
5. In the publish window, review your changes under **Review & select**.
   - You’ll see any new, modified, or removed templates listed here.
   - Templates with no changes are grouped under Unchanged.
6. Click **Next**, edit the template details under **Set details**, and then click **Publish**.

Template users can [review design updates](https://help.figma.com/hc/en-us/articles/33799043961879) in their files to accept any changes made to templates they are using in their files.

## Unpublish a template

1. Open the original template file.
2. Click on the dropdown menu next to the file name.
3. Hover over **Manage template > Unpublish template**.
4. Click **Unpublish** to confirm.

The template will no longer appear in the template picker for your team or organization. Existing files that use the template will not be impacted, but new assets using the template can no longer be added to the file.

Note: Moving the original template file back to your drafts will also unpublish the template.

## Disable custom templates

Organization admins can prevent members from publishing custom templates.

1. Open the file browser.
2. Select **Admin** in the sidebar.
3. Select the **Settings** tab.
4. Scroll down to the **Other** section, then turn off the **Template publishing** toggle.

Custom Figma Buzz templates created in your organization will no longer be displayed in the template picker. If you decide to re-enable custom templates, previously created templates will be restored to the template picker.
