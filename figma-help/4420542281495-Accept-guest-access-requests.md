## 내용

Who can use this feature

Supported on the [Enterprise plan](https://help.figma.com/hc/en-us/articles/360040328273).

When the [guest membership setting](https://help.figma.com/hc/en-us/articles/4410793238167) is set to require approval, admins need to accept new guests added or invited to the organization.

Guests only require approval when they first join the organization. Once accepted, guests can be invited to other resources in the organization without admin approval.

### Who can accept guest requests

Only admins can accept new guests. Figma uses the inviter’s workspace, or the location of the file, to determine who gets the request.

- If the inviter is assigned to a workspace, or the file lives in a workspace, workspace admins can accept or ignore the request.
- If the inviter isn’t assigned to a workspace, or the file lives outside a workspace, an organization admin can accept or ignore the request

## Accept requests

Admins can only accept or ignore the request. If a request is ignored, the guest won’t be able to send another request.

1. Open email / in-app notification
2. Click **View request** to review the request.
3. View details about the request, including:
   - Guest name and email address (matches their Figma account)
   - Organization name
   - Inviter's name, access level (view/edit), resource name
4. Select **Dismiss** to ignore the request, or to add the guest via your identity provider.
5. Select **Add** to accept the request.

Note: Guests can have the same seat types and team permissions as members.

- **Seats**: Guests can have any [seat type](https://help.figma.com/hc/en-us/articles/360039960434): Full, Dev, Collab, or View. [Seat approval settings](https://help.figma.com/hc/en-us/articles/4414038570007) apply to guests.
- **Permissions**: You choose their permissions when you invite them to a file, project, or team. You can give them different permissions on different resources.
