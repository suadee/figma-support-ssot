## 내용

Before you start

Who can use this feature

 

Users on [any Figma team or plan](https://help.figma.com/hc/en-us/articles/360040328273-Choose-a-Figma-Plan) can embed files and prototypes.

Users with `can view access` to the file can interact with a file embed.

Users with can edit access to the file can interact with a prototype embed.

Want to embed a file or prototype? Check out our [Embed files and prototypes](https://help.figma.com/hc/en-us/articles/360039827134) article.

You can add live Figma files or prototypes into other websites or applications. Use embeds to:

- Link design files to product specifications
- Add relevant resources to project or task management tools
- Showcase use cases and examples within documentation
- Share presentations, templates, or resources
- Extend the functionality a Design System

Embeds appear alongside any existing content, making them a seamless addition to any website or application.

# View embeds

You can interact with each embed based on your [team](https://help.figma.com/hc/en-us/articles/360039970673) or [Organization permissions](https://help.figma.com/hc/en-us/articles/360039960434), and the file's link sharing settings.

**If link sharing is on (Public file):** view the embed regardless of your permissions on the file.

**If link sharing is off (Private file):** you need to have `can view` or `can edit` access to the file to view the embed. 

Learn more about link sharing:

- **Starter, Education, and Professional teams:** [Share public and private files](https://help.figma.com/hc/en-us/articles/360040531773)
- **Organizations:** [Share files in an Organization](https://help.figma.com/hc/en-us/articles/360040044834)

## Authentication

The first time you access an embed in a given location, such as the first time viewing an embed on a specific website or using one of the apps where Figma embeds are supported, Figma may prompt you to allow access to cookies. Figma requires this permission in order to load the embedded file or prototype. If cookies are inaccessible, or if you prefer not to allow cookies, you can use the **open this file in Figma** link.

![Figma prompt requesting browser cookie access to verify permissions for viewing an embedded file. Options to allow or open in Figma.](https://help.figma.com/hc/article_attachments/26808296257303)

If you're **already logged in to Figma** and you’ve already **allowed cookies**, you can see and interact with any embeds you have access to.

If you're **not logged in to Figma**—or are on a website that blocks browser cookies—Figma will prompt you to log in before you can interact with the embed. You may be redirected to the browser to log in.

![Log in screen for accessing a Figma file, featuring a prompt with login button and account creation link.](https://help.figma.com/hc/article_attachments/26000622737943)

On some occasions, Figma may need to perform a security verification. If you are prompted, click **Verify to continue** and complete the security challenge.

![Security verification prompt with "Verify to continue" button to access Figma embed.](https://help.figma.com/hc/article_attachments/26808296262551)

If the file **has a password**, you need to enter the password when prompted in order to use the file.

![Password prompt screen for accessing a Figma file with an "Enter Password" button.](https://help.figma.com/hc/article_attachments/26000622744215)

If you don't have access, you'll see another message on the embed. We recommend reaching out to the file owner so they can add or update your access.

- **Figma Organization:** Figma will show a **You don't have access to the file** message.
- **Professional teams:** Figma will show a **File not found** message.

Logged in with the wrong email address? Figma will display the email address you are currently logged in with. Click the Sign out button to log out of the account.

## File embeds

File embeds are view-only. There is no way to select individual layers or make any edits to the file. To edit a file, click on the file name to open it in Figma.

In a file embed, you can:

1. Use the viewport to view the file and pan around the canvas
2. Select which page in the embedded file you want to view
3. Enter full screen mode
4. Zoom in or out in the viewport
5. Click on the file name to open the file in Figma
6. View the file's [link sharing settings](https://help.figma.com/hc/en-us/articles/360040531773)
7. See when the file was last edited

![File embed feature with options to select pages, zoom, enter full screen, view edit history, and link sharing settings.](https://help.figma.com/hc/article_attachments/26000642048919)

## Prototype embeds

Interact with a prototype embed as if you are viewing the prototype in [presentation view](https://help.figma.com/hc/en-us/articles/360040318013).

In a prototype embed, you can:

1. View and interact with the hotspots in the viewport
2. Select the prototype flow you want to view
3. Enter full screen mode
4. Click on the file name to open the file in Figma
5. Use the arrow keys to move between screens in the prototype, and restart the prototype
6. View the file's [link sharing settings](https://help.figma.com/hc/en-us/articles/360040531773)
7. See when the file was last edited

![Figma prototype embed showing interactive viewport with selectable flows, fullscreen option, and link sharing settings.](https://help.figma.com/hc/article_attachments/26000622748823)
