## 내용

**Note:** The Figma for Jira integration is co-owned by Figma and Atlassian. For any issues with Jira functionality, including the Design section in Jira issue views, please contact Atlassian Support, as these components are owned and maintained by Atlassian.

#### **Who can use this feature**

- Available with the Figma for Jira app.
- Requires permission to view the linked Figma file and access to the Jira issue.

[Jira Software](https://www.atlassian.com/software/jira) is a project and issue tracking tool for agile teams. This article covers connecting Figma to Jira Software.

Connect Figma to Jira Software to bring live design context to your team’s backlog and sprints. You can link files or prototypes to issues, preview designs without switching tools, and see real‑time design status in Jira. The integration now uses automatic file‑level webhooks, so when you link a Figma file, Jira reflects design updates and Dev Mode status with no admin setup required.

You will need to [create a Jira Software account](https://www.atlassian.com/software/jira/free) to use this integration. It's not possible to add embeds to [Jira Align](https://www.atlassian.com/software/jira/align) or [Jira Service Desk](https://www.atlassian.com/software/jira/service-desk) using this integration.

[Learn more about embedding Figma files in other tools](https://help.figma.com/hc/en-us/articles/360039827134).

## Install the Figma for Jira app

Before you can embed files and prototypes in Jira Software, you will need to add the Figma app to your Jira account.

You can do this from anywhere inside your Jira account.

1. In the sidebar under **Apps**, click **Explore more apps**.
2. Search for ‘Figma for Jira’ and click on the app in the results.
3. Click **Get it now** in the top-right corner of the listing to add the app to your account.

Once you've installed the Figma for Jira app, your Jira issues will have a **Designs**section. This is where you can add and view Figma Live Embeds.

## Add a file or prototype to Jira

You can embed any file or prototype, regardless of the file's [link sharing settings](https://help.figma.com/hc/en-us/articles/360040531773). Collaborators can interact with Figma Live Embeds based on:

- Their [team](https://help.figma.com/hc/en-us/articles/360039970673) or [Organization permissions](https://help.figma.com/hc/en-us/articles/360039960434)
- The file's sharing settings

## Link a design file in Jira Software

You can link multiple Figma files to issues and get notified when designs are updated, without leaving Jira.

**Tip:** While in Board view, you’ll see a **design linked** icon on Jira issues that have a linked Figma Design file. Hover over the design icon to see information about the design and when it was last updated, access the Figma link, or view the design directly in the expanded view in Jira.![Jira board displaying linked Figma design with a "Ready for Dev" status tag and design update notification.](https://help.figma.com/hc/article_attachments/21161264631447)

When you add a design to an issue, the Jira link is automatically added under the Figma file’s **Dev resources** in Dev Mode. You can click on the Dev resource link to open the issue in the Jira plugin, without leaving Figma.  
[Learn more about using Dev resources in Dev Mode →](https://help.figma.com/hc/en-us/articles/15023231995927-Link-Dev-resources-to-layers-in-Dev-Mode)

1. Open the Jira issue you want to add an embed to.
2. If the **Designs** section isn't shown, click **Add design** under the title. You can also click **Sync**to view any previously linked designs.
3. Paste the Figma URL in the field and click **Link**.
4. You can now [view the file](https://help.figma.com/hc/en-us/articles/360051741274) without leaving Jira. Click on the thumbnail to expand the design preview. Clicking on a prototype will open the file in a new tab. ![Jira issue interface showing a linked Figma design with integration status and project details.](https://help.figma.com/hc/article_attachments/19619789757335)![Design preview window showing an integrations landing design within Jira, with options to inspect in Figma or view as prototype.](https://help.figma.com/hc/article_attachments/19619789760279)
5. Click  in the **Designs** section to link multiple files.![Jira issue view with linked Figma designs, showing details like assignee, priority, and status.](https://help.figma.com/hc/article_attachments/19619805607191)

## Link a design file with the Jira plugin

Get full context when turning designs to code with the Jira plugin. With the plugin, you can link Jira issues to Figma pages or layers so you always have the relevant information you need.

Use the Jira plugin to:

- Import and link existing Jira issues to pages or layers in a Figma file, so that developers have all the relevant context they need easily accessible.
- View linked Jira issues directly in Figma, to reduce context switching during design implementation.
- Create new Jira issues directly from Figma, to stay in flow even if you realize new issues need to be created as part of planning or handoff.

When you link a design to an issue using the Jira plugin, a link to the design file is automatically added to the Jira issue, under the **Designs** section.

[Check out the Jira plugin in the Figma Community →](https://www.figma.com/community/plugin/1220802563996996107)

To use the Jira plugin in Figma:

1. Run the Jira plugin from your design file. The plugin is available in both Design Mode and [Dev Mode](https://help.figma.com/hc/en-us/articles/15023124644247-Guide-to-Dev-Mode).  
   [Learn more about using plugins in files →](https://help.figma.com/hc/en-us/articles/360042532714-Use-plugins-in-files)
2. Select the specific page or layer you want to embed.
3. Click **Create issue** to add a new issue in Jira, or **Link issue** to connect an existing Jira issue to your selected page or layer.
4. Once an issue is linked, you’ll be able to view its details within the Jira plugin.

**Note:** When you use the plugin to link an issue to a design, the issue is also added as a link under Dev resources in Dev Mode for added discovery.

[Learn more about using Dev resources in Dev Mode ->](https://help.figma.com/hc/en-us/articles/15023231995927-Link-Dev-resources-to-layers-in-Dev-Mode)

## Create issues from the Jira widget

Use the Jira widget to pull Jira projects and all of their associated issues into the canvas to discuss and update with your team. Adjust assignee, priority, status, and issue type right in Figma or FigJam, while maintaining your source of truth in Jira. You can easily capture them as new to-do’s as issues in Jira, and even link associated designs from Figma.

[Check out the Jira widget in the Figma Community ->](https://www.figma.com/community/widget/1094001923188252679)

### In Figma Design

When you run the Jira widget from a design file, you can either create an issue from the design, or import an existing issue or project.

[Learn more about using widgets in files →](https://help.figma.com/hc/en-us/articles/4410047809431-Use-widgets-in-files)

To create an issue from a design:

1. Click **Create issue from design** in the widget.
2. Add a brief **Summary** of your issue.
3. Select where to add the design from. You can add designs from a selected frame, your current page, or by pasting in a Figma link.
4. Select a **Project** from the dropdown to add the new issue.
5. Select an **Issue type** from the dropdown.
6. Click **Create** to generate the issue. You’ll see the issue as a draggable object in the file.

### In FigJam

When you run the Jira widget from a FigJam file, you can either turn stickies into issues, or import an existing issue or project.

To turn stickies into issues:

1. Select a single sticky or multiple stickies to turn into issues.
2. Click **Turn stickies into issues** in the widget.
3. Select a **Project** from the dropdown to add the new issue.
4. Select an **Issue type** from the dropdown.
5. Click **Convert stickies** to generate the issues. You’ll see the sticky turn into a draggable issue in the canvas.

## View Design status in Jira

You can see real-time design status in Jira after you link a Figma file to an issue with no admin setup required. The Figma for Jira app automatically adds file-level webhooks so status appears in near real time.

**To view status:**

1. Open a Jira issue with a linked Figma file in the **Designs** section.
   - If the file isn’t linked yet, paste the Figma file URL and click **Link**.
2. Look for the status next to the design thumbnail:
   - **Design updated** - The linked Figma file has changed since you last viewed this issue.
   - **Ready for dev** - Mirrors the file’s [Dev Mode status](https://help.figma.com/hc/en-us/articles/26781702258583-Dev-Mode-statuses-and-notifications#h_01J96T2MJRC6T23W0N77D20RGM) in Figma.
3. Click the thumbnail to expand the preview, or open in Figma for full context.

**Note:** For the **Design updated** tag to appear in the Jira issue, all active users must first close any browser tabs that have the linked Figma Design file.

### How updates work

When you link a Figma file, the integration automatically registers file-level webhooks. Status updates in Jira when the file’s Dev Mode status changes or when there are new changes to the file.

No team URL or admin configuration is required.

### Troubleshooting

- Make sure you linked a file (not just a page link without the file).
- Confirm you have permission to view the Figma file.
- Click Sync in the Designs section (if shown) to refresh, or unlink and re-link the file if status doesn’t appear on an older issue link.

## Automate workflows with linked designs

Save time by automating manual processes to improve collaboration between
design and development teams. Use design automation in Jira to notify developers
of changes to designs, transition issue statuses, and easily add labels to
issues with designs.

![Automation panel in Jira showing a rule setup to add a label when a design is linked to an issue.](https://help.figma.com/hc/article_attachments/21161403501079)

To set up automations with linked designs in a Jira project:

1. Go to **Project settings** > **Automation** in the left sidebar.
2. Click on the **Templates** tab.
3. In the **Design** section, select the automation you want to set up.
4. Click **Turn on rule**. Next to the rule name, you’ll see its status change to **Enabled**.

You can disable or delete an automation from the **Rules** tab under **Automation**.

[Learn more about automating tasks and processes in Jira →](https://www.atlassian.com/software/jira/guides/automation/tutorials#manage-automation-rules)

## Remove Figma from Jira

You can remove a single embed, or uninstall the Figma app to remove the app and all Figma embeds.

## Remove an embed

If you want to delete a specific file or prototype, you can remove it from the issue in Jira.

1. Open the issue you want to remove the embed from.
2. Click **Remove design** next to the embed.
3. Jira will remove that embed from the issue.

## Remove the Figma for Jira app

If you want to disconnect your Figma account from Jira, you can uninstall the Figma app. This will also remove any file or prototype embeds from your board.

1. Open the board in Jira.
2. Click **Apps** in the menu bar and select **Manage your apps**. ![Jira Software Kanban board view with tasks listed under different stages such as Inbox, In Progress, and Done.](https://help.figma.com/hc/article_attachments/360070674374)
3. Click on the arrow next to the Figma application to expand.
4. Click the **Uninstall** button:

   ![Uninstall option for Figma for JIRA Cloud in the Manage Apps section of Jira Software.](https://help.figma.com/hc/article_attachments/360070674514)
