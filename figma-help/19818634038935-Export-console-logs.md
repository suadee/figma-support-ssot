## 내용

**What:** Use the developer tools to view and capture any errors in the *console* log.

**Why:** The console log gives us access to a list of actions the browser has performed. This will highlight any error messages that appeared during the process.

## Open developer tools

Open the **Developer tools** using the keyboard shortcut. This works in both the Figma desktop app and in the browser.

- **Mac:**  `Command` `Option` `I`
- **Windows:**  `Ctrl` `Alt` `I` *(on desktop app)* or `F12` *(on browsers)*

Or, head to **Help** in the menu bar and selecting **Toggle Developer Tools:**

![Menu with "Toggle Developer Tools" highlighted for accessing developer tools in Figma via the menu bar.](https://s3.amazonaws.com/helpscout.net/docs/assets/5aa962fe2c7d3a2c4983093d/images/5d77319404286364bc8ee9b5/file-zMrOVkox1o.png)

## Export console log

1. From the Developer Tools window, select the **Console** tab:![Developer Tools window showing HTML code and styles pane for debugging errors in the console.](https://s3.amazonaws.com/helpscout.net/docs/assets/5aa962fe2c7d3a2c4983093d/images/5d77319c2c7d3a7e9ae0f1f7/file-y6aIrFdgwo.png)
2. Any Console errors will be shown in red. Right-click in the console and choose **Save as** from the options.![Developer Tools window with Console tab open, displaying red error messages related to access issues and status 429.](https://s3.amazonaws.com/helpscout.net/docs/assets/5aa962fe2c7d3a2c4983093d/images/5d7731af2c7d3a7e9ae0f1f9/file-8RUkin7ESG.png)

   **Note:** We recommend **saving a copy** of the log. This allows us to view the errors in more detail and is preferable to taking a screenshot.  
     
   You may see console errors relating to "api-iam.intercom.io". These do not impact performance or indicate an error with Figma. You can safely ignore these.
3. Attach the File(s) to your conversation with Figma Support.
